# APEX Housing вҖ” Frontend Architecture Blueprint

## TWO SEPARATE APPS

| App | Package | Users | Platform |
|-----|---------|-------|----------|
| **APEX Housing** | `com.apexhousing.user` | Tenants + Landlords + Agents | Mobile (iOS/Android) |
| **APEX Admin** | `com.apexhousing.admin` | Admins + Super Admins | Web-first (Next.js), mobile secondary |

Both apps share the **same backend API** (197 endpoints). They use the same auth system вҖ” an admin user logs in with the same credentials, but the admin app checks `user.role === "ADMIN"` and shows admin UI instead.

---

## Tech Stack (Recommended)

| Layer | APEX Housing (User App) | APEX Admin |
|-------|------------------------|------------|
| Framework | React Native (Expo) | Next.js 14 (App Router) |
| Navigation | React Navigation (bottom tabs + stacks) | Next.js file-based routing |
| State | Zustand + TanStack Query | Zustand + TanStack Query |
| Styling | NativeWind (Tailwind for RN) | Tailwind CSS |
| Forms | React Hook Form + Zod | React Hook Form + Zod |
| Signature | react-native-signature-capture | signature_pad (web canvas) |
| Maps | react-native-maps / Mapbox GL | Mapbox GL JS / Leaflet |
| Payments | Paystack React Native SDK | Paystack inline JS |
| Push | Firebase Cloud Messaging (FCM) | FCM + Email alerts |
| Storage | Supabase Storage REST API | Supabase Storage REST API |
| Real-time | Supabase Realtime | Supabase Realtime + WebSocket |

## KEY USER FLOWS

### Flow 1: New Tenant вҖ” First Booking (User App)
```
Register вҶ’ OTP вҶ’ Home вҶ’ Search вҶ’ Property Detail вҶ’ Book Now
  вҶ’ Check Signature вҶ’ [No signature] вҶ’ Signature Pad Modal
  вҶ’ Draw Signature вҶ’ Saved to account
  вҶ’ Confirm & Pay вҶ’ Paystack вҶ’ Payment Success
  вҶ’ Booking Created вҶ’ Escrow Created
  вҶ’ Confirmation screen
```

### Flow 2: Returning Tenant вҖ” Second Booking (User App)
```
Login вҶ’ Home вҶ’ Search вҶ’ Property Detail вҶ’ Book Now
  вҶ’ Check Signature вҶ’ [Has signature] вҶ’ Auto-applied вң“
  вҶ’ No signature step needed
  вҶ’ Confirm & Pay вҶ’ Done (faster!)
```

### Flow 3: Tenant Inspection Flow (User App)
```
Booking Confirmed вҶ’ Move-in Confirmed вҶ’ 30hr Timer Starts
  вҶ’ Timer Screen (countdown)
  вҶ’ Satisfied вҶ’ Funds Released вҶ’ Review Prompt
  OR
  вҶ’ Not Satisfied вҶ’ Dispute Opened вҶ’ Evidence Upload
  вҶ’ Admin Reviews in Admin App вҶ’ Resolution
```

### Flow 4: Landlord вҖ” List Property (User App)
```
Switch to Landlord вҶ’ Dashboard вҶ’ Create Listing
  вҶ’ Step 1: Type & Info
  вҶ’ Step 2: Location (map pin)
  вҶ’ Step 3: Pricing
  вҶ’ Step 4: Images & Video
  вҶ’ Step 5: Features & Amenities
  вҶ’ Step 6: Availability Window
  вҶ’ Step 7: Terms & Signature
  вҶ’ Submit вҶ’ Pending Approval
  вҶ’ Admin Approves in Admin App вҶ’ Property Active
```

### Flow 5: Landlord вҖ” Withdraw Earnings (User App)
```
Dashboard вҶ’ Wallet вҶ’ Withdraw Funds
  вҶ’ Enter Amount вҶ’ Select Bank Account
  вҶ’ Confirm вҶ’ Paystack Transfer
  вҶ’ Business Day Check
  вҶ’ Funds Transferred вҶ’ Notification
```

### Flow 6: Admin вҖ” Approve Property (Admin App)
```
Admin Login вҶ’ Dashboard вҶ’ "5 Properties to approve" card
  вҶ’ Pending Properties List вҶ’ Tap property
  вҶ’ Review details (images, pricing, terms, agent signature)
  вҶ’ [Approve вң…] or [Reject вқҢ with reason]
  вҶ’ Property status changes вҶ’ Landlord notified in User App
```

### Flow 7: Admin вҖ” Review KYC (Admin App)
```
Admin Login вҶ’ Dashboard вҶ’ "3 KYC to review" card
  вҶ’ Pending KYC List вҶ’ Tap user
  вҶ’ View ID document image + selfie
  вҶ’ OCR results displayed (name, number, expiry)
  вҶ’ Face match result
  вҶ’ [Approve вң…] or [Reject вқҢ with reason]
  вҶ’ User's KYC status updated вҶ’ User notified in User App
```

### Flow 8: Admin вҖ” Resolve Dispute (Admin App)
```
Admin Login вҶ’ Disputes вҶ’ Open dispute
  вҶ’ Read issue description + evidence files
  вҶ’ Message thread with tenant + landlord
  вҶ’ [Resolve] вҶ’ Choose action:
```
